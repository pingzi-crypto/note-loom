# Note Loom

[English](./README.md) | 简体中文

Note Loom 是一个把随手记录的笔记，按你已有模板生成结构化成品笔记的 Obsidian 插件。你可以随心所欲地记录，生成前再预览、调整和确认。

## 快速开始

1. 导入模板
   打开 `设置 -> 第三方插件 -> Note Loom`，选择模板根目录，导入要使用的 Markdown 模板。

2. 配置模板
   在模板列表中点击 `Edit`，确认输出目录、文件名字段、字段别名和需要生成的区块。简单模板通常保留默认配置即可。

3. 打开源笔记
   打开一篇 Inbox 随笔、会议速记、读书摘录或项目记录。

4. 开始生成
   打开 Obsidian 命令面板，搜索并执行 `生成结构化笔记`；也可以点击左侧 Ribbon 图标，选择要使用的模板。

5. 预览并确认
   在生成弹窗中检查字段、区块、输出路径和预览内容。需要时手动调整，然后生成成品笔记。

## 一个最小例子

源笔记：

```text
标题：渐进阅读法
主题：如何把一本书拆成可复用的概念卡片
核心观点：先抓问题，再整理观点，最后沉淀行动。
下一步：补一张概念卡片，连接到阅读索引。
```

模板：

```md
---
title: {{标题}}
tags:
---

# {{标题}}

## 主题
{{主题}}

## 核心观点
{{核心观点}}

## 下一步
{{下一步}}
```

生成结果：

```md
---
title: 渐进阅读法
tags:
---

# 渐进阅读法

## 主题
如何把一本书拆成可复用的概念卡片

## 核心观点
先抓问题，再整理观点，最后沉淀行动。

## 下一步
补一张概念卡片，连接到阅读索引。
```

## 它能处理什么

- 面向静态和半动态 Markdown 模板：固定标题、段落、列表、字段块、表格、任务清单这类结构最适合使用。
- 使用 `{{字段名}}` 作为主要占位符，并支持写入 frontmatter、正文重复字段和 Dataview inline fields。
- 支持常见结构区块：单记录字段块、分组字段块、重复条目、Markdown 表格、任务列表和 checkbox 选项组。
- 兼容 Templater / Dataview 的常见用法：简单日期等 Templater 内容可处理，Dataview 查询和脚本展示区会保留并提示复核。
- 从当前打开的 Markdown 笔记生成新笔记，按字段名、字段别名、区块别名和清晰的标点边界匹配内容。
- 生成前可以预览、修改、禁用字段，并确认输出路径、文件名和索引写入方式。

## 使用建议

- 字段名或别名后面加上常见标点，匹配最稳定。例如：`标题：渐进阅读法`、`下一步，补一张概念卡片`。
- 如果源笔记没有写字段名，也没有配置对应别名，插件不会只靠语义去猜这个字段。
- 区块内容建议写出区块标题或区块别名；表格、清单、重复记录这类结构，尽量保留清楚的条目边界。
- 第一次使用新模板时，先生成一两篇测试笔记，确认效果后再用于正式流程。

## 适合哪些场景

- 读书和文章摘录：先把触动你的段落、问题和下一步随手写下，之后生成到读书笔记、摘要或概念卡模板。
- 会议和讨论记录：先快速记结论、行动项、负责人和风险，之后生成到会议纪要或项目追踪模板。
- 项目推进和复盘：先记录进展、卡点、证据和下一步，之后生成到项目状态、复盘或 checklist 模板。
- 日记和时间账本：先按一天的真实节奏记录事件、情绪、时间段和明日安排，之后生成到日记、时间记录或习惯追踪模板。

## 为什么值得用

很多人的 Obsidian 不是缺模板，而是缺一条从 Inbox 到成品笔记的路径。

Note Loom 把松散记录送进你已有的知识库、项目库、会议库或日记系统。重复搬运、字段对齐和结构整理交给插件，判断、取舍和确认留给你。

结果是：你不用再靠意志力清空 Inbox，也不用把宝贵的整理时间浪费在机械搬运上。
